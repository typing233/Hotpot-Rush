# 🍲 火锅冲冲冲 - Hotpot Rush

一款考验眼手协调能力的休闲解压闯关游戏。

## 🎮 游戏特色

### 🎯 核心玩法
- **传送带食材系统**: 屏幕四周环绕单向匀速滑动的传送带，不断随机刷新多种食材
- **订单匹配机制**: 屏幕顶部显示当前关卡的火锅订单，点击符合条件的食材触发抛物线动画飞入锅中
- **倒计时挑战**: 在时间耗尽前凑齐订单所需食材数量即可通关

### ⚠️ 干扰机制
- **坏食材 (💀)**: 点击会扣除分数和时间
- **炸弹 (💣)**: 点击造成大量扣分和时间惩罚

### 🎁 增益道具
- **冰冻道具 (❄️)**: 让传送带速度减半，持续5秒
- **漏勺道具 (🥄)**: 自动吸附符合订单的食材，持续5秒

### ⚡ 连击系统
- 连续正确点击食材触发连击加成
- 连击倍数最高可达5倍
- 2秒内无正确点击连击重置

### 💰 金币与商店
- 通关获得金币奖励
- 金币可用于：
  - 🪑 解锁升级桌椅（基础木桌 → 红木餐桌 → 大理石桌 → 豪华金桌）
  - 🖼️ 解锁升级墙纸（白色墙壁 → 温馨墙纸 → 古典壁画 → 霓虹夜景）
  - 🍲 解锁不同造型火锅（普通单锅 → 鸳鸯锅 → 九宫格 → 四宫格）

### 📚 菜品图鉴
- 收集11种不同食材
- 包含普通、稀有、史诗三种稀有度

### 🔊 音效系统
- 水沸腾声（每3秒循环）
- 点击食材的"啵唧"声
- 连击音效
- 道具获得音效
- 通关/失败音乐

## 🚀 启动方式

### 1. 启动后端服务

```bash
cd /home/ubuntu/Hotpot-Rush/backend

# 创建虚拟环境（首次运行）
python3 -m venv venv

# 激活虚拟环境
source venv/bin/activate

# 安装依赖（首次运行）
pip install -r requirements.txt

# 启动服务器
python app.py
```

后端服务将在 **端口5000** 运行。

### 2. 启动前端服务

```bash
cd /home/ubuntu/Hotpot-Rush/frontend

# 启动简单HTTP服务器
python3 -m http.server 8080
```

前端服务将在 **端口8080** 运行。

### 3. 访问游戏

打开浏览器访问：`http://localhost:8080`

### 玩家数据
- `GET /api/game-data` - 获取玩家数据
- `POST /api/complete-level` - 完成关卡

### 关卡
- `GET /api/levels` - 获取所有关卡列表
- `GET /api/levels/<level_id>` - 获取单个关卡详情

### 商店
- `POST /api/shop/buy` - 购买商品
- `POST /api/shop/select` - 选择已解锁商品

### 图鉴
- `GET /api/collection` - 获取菜品图鉴

## 🎮 游戏机制详解

### 传送带路径
食材沿屏幕四周顺时针移动：
```
    → → → → → → →
  ↑                 ↓
  ↑                 ↓
  ↑                 ↓
    ← ← ← ← ← ← ←
```

### 分数计算
```
基础分数 = 食材基础分 * 连击倍数
连击倍数 = 1.5 ^ (连击数 - 1) （最高5倍）
```

### 道具效果
- **冰冻**: 传送带速度 = 原速度 * 0.5
- **漏勺**: 自动检测并收集符合订单的食材

### 数据保存
游戏数据保存在 `backend/data/game_data.json`，包括：
- 玩家金币数量
- 已完成关卡
- 商店购买记录
- 已选择的装饰
- 收集的菜品图鉴

## 🎯 操作说明

1. **开始游戏**: 点击主菜单"开始游戏" → 选择关卡
2. **游戏操作**: 点击传送带上符合订单的食材
3. **暂停**: 点击右上角暂停按钮
4. **使用金币**: 返回主菜单 → 进入商店购买装饰
5. **查看图鉴**: 返回主菜单 → 进入菜品图鉴

## 🐛 已知问题与注意事项

1. 首次运行需要创建Python虚拟环境并安装依赖
2. 浏览器需要支持Web Audio API（现代浏览器均可）
3. 建议在全屏模式下游玩以获得最佳体验
4. 游戏数据本地存储，清除浏览器缓存不影响（数据存在后端）

## 📝 开发说明

### 修改关卡配置
编辑 `backend/app.py` 中的 `LEVELS` 列表可自定义：
- 时间限制
- 订单内容
- 食材种类
- 传送带速度
- 生成频率
- 奖励金币

### 添加新食材
修改 `ALL_DISHES` 列表添加新的菜品图鉴条目。

### 修改样式
编辑 `frontend/styles.css` 自定义游戏外观。
